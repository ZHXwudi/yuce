# Streamlit 页面分析模块说明

本文档解释当前 Streamlit 展示页中“诊断总览、信号分析、模型评估、项目展示”各部分的含义、取值方式、计算公式和通俗解释。

当前数据文件为 `price_input_foreign.csv`，字段包含日期、时间、电价。页面把“电价”作为一个可替换的工业时序信号样例来处理。后续如果换成真实设备数据，可以把 `price` 理解为振动、电流、温度、转速、压力等设备状态量。

## 1. 数据预处理

### 1.1 原始字段

原始 CSV 字段：

| 字段 | 页面内部字段 | 含义 |
|---|---|---|
| 日 | `day` | 日期中的日 |
| 月 | `month` | 日期中的月 |
| 日期 | `date_text` | 日期文本 |
| 时间 | `time_text` | 小时时间 |
| 电价 | `price` | 当前页面使用的状态信号 |

### 1.2 时间解析

页面将 `日期 + 时间` 合并为 `timestamp`：

```text
timestamp = parse_datetime(date_text + " " + time_text)
```

日期按 `日/月/年 小时:分钟` 解析，例如：

```text
31/12/2025 23:00
```

页面会删除：

- 全空行
- 没有日期、时间或电价的行
- 无法解析为时间的行
- 重复时间戳

### 1.3 派生时间字段

从 `timestamp` 中提取：

```text
year = 年份
month = 月份
day = 日期中的日
hour = 小时
weekday = 星期几
date = 日期
```

通俗解释：这一步是把原始表格整理成可分析的时间序列，让后面的异常识别、周期分析、聚类和预测都能按时间顺序计算。

## 2. 诊断总览

诊断总览是页面第一屏，回答三个问题：

1. 数据整体情况怎样？
2. 哪些时间点可能异常？
3. 异常主要由什么原因驱动？

### 2.1 顶部指标卡

| 指标 | 取值方式 | 通俗解释 |
|---|---|---|
| 有效样本 | `len(df_view)` | 当前筛选年份下有多少条有效小时数据 |
| 均值 | `mean(price)` | 整体平均水平 |
| 峰值 | `max(price)` | 最高信号值 |
| 谷值 | `min(price)` | 最低信号值 |
| 异常小时 | `sum(is_anomaly)` | 被系统判定为异常的小时数 |
| 负值小时 | `sum(price < 0)` | 信号值小于 0 的小时数 |

如果换成真实设备数据：

- 峰值可能代表振动冲击峰值、电流尖峰、温度过高等。
- 谷值或负值可能代表传感器异常、倒挂、回流、设备状态突变等。
- 异常小时代表需要重点复核的运行时段。

### 2.2 鲁棒 Z 分数

页面没有直接用普通均值和标准差做异常识别，而是使用鲁棒 Z 分数。

公式：

```text
median_x = median(x)
MAD = median(|x - median_x|)
scale = 1.4826 * MAD
robust_z = (x - median_x) / scale
```

其中：

- `median_x` 是中位数。
- `MAD` 是中位绝对偏差。
- `1.4826` 是把 MAD 调整到接近标准差量级的系数。

如果 `scale` 太小，页面会退回使用普通标准差：

```text
scale = std(x)
```

通俗解释：普通均值和标准差容易被极端异常值带偏。鲁棒 Z 分数更像“拿大多数正常点作为参照”，判断某个点偏离正常水平有多远。

### 2.3 异常诊断特征

页面为每个小时计算 4 类异常特征。

#### 2.3.1 幅值异常：`z_price`

```text
z_price = |robust_z(price)|
```

含义：当前信号值本身是否过高或过低。

通俗解释：比如振动值突然很高，或者电价出现极高/极低，都可能被这个指标抓到。

#### 2.3.2 小时突变：`z_change`

先计算一小时差分：

```text
price_diff_1h[t] = price[t] - price[t-1]
```

再计算：

```text
z_change = |robust_z(price_diff_1h)|
```

含义：相邻小时之间变化是否剧烈。

通俗解释：如果设备状态不是慢慢变化，而是在短时间内突然跳变，这个指标会升高。

#### 2.3.3 短窗残差：`z_residual`

先计算 24 小时滚动均值：

```text
rolling_mean_24h[t] = mean(price[t-23], ..., price[t])
```

再计算残差：

```text
residual_24h[t] = price[t] - rolling_mean_24h[t]
```

最后计算：

```text
z_residual = |robust_z(residual_24h)|
```

含义：当前点相对最近 24 小时平均水平偏离多少。

通俗解释：如果一个设备最近一天都比较稳定，但某个小时突然偏离最近状态，这个指标会升高。

#### 2.3.4 季节模式偏离：`z_pattern`

页面按“月份 + 小时”建立一个典型模式基线：

```text
pattern_median[month, hour] = median(price in same month and same hour)
```

当前点与典型模式的偏差：

```text
pattern_residual[t] = price[t] - pattern_median[month[t], hour[t]]
```

再计算：

```text
z_pattern = |robust_z(pattern_residual)|
```

含义：当前点是否偏离同月份、同小时的常见模式。

通俗解释：比如每天晚上某个设备负荷通常较高，这是正常规律；但如果某天同一小时明显不同，就可能是工况异常。

### 2.4 综合异常评分

页面把 4 类异常特征加权合成为一个诊断评分：

```text
diagnostic_score =
    0.35 * z_price
  + 0.25 * z_change
  + 0.25 * z_residual
  + 0.15 * z_pattern
```

权重含义：

| 特征 | 权重 | 为什么这样设 |
|---|---:|---|
| 幅值异常 `z_price` | 0.35 | 直接越限最重要 |
| 小时突变 `z_change` | 0.25 | 突变常对应冲击或状态切换 |
| 短窗残差 `z_residual` | 0.25 | 反映短期漂移和局部异常 |
| 模式偏离 `z_pattern` | 0.15 | 反映季节/小时规律偏离 |

通俗解释：页面不是只看“值大不大”，而是同时看“变化快不快、相对最近状态是否异常、是否偏离常见周期模式”。

### 2.5 异常判定

页面用侧边栏的“异常评分分位阈值”决定异常线，默认是 98 分位。

公式：

```text
threshold = percentile(diagnostic_score, anomaly_percentile)
```

默认：

```text
threshold = percentile(diagnostic_score, 98)
```

异常判定：

```text
is_anomaly = diagnostic_score >= threshold OR price < 0
```

通俗解释：

- 如果一个点的综合异常评分排在最异常的前 2%，就标记为异常。
- 如果信号值小于 0，也直接标记为异常。对于电价数据，负值电价是特殊现象；对于设备数据，负值可能代表传感器、采集或业务逻辑异常。

### 2.6 诊断等级

页面进一步划分异常等级：

```text
high_cut = percentile(diagnostic_score, 99.4)
mid_cut = percentile(diagnostic_score, 98.2)
```

等级规则：

```text
if diagnostic_score >= high_cut:
    severity = "高风险"
elif diagnostic_score >= mid_cut:
    severity = "中风险"
elif price < 0:
    severity = "中风险"
else:
    severity = "观察"
```

通俗解释：

- 高风险：最极端的一小部分点，需要优先检查。
- 中风险：比较异常，但可能需要结合上下文确认。
- 观察：有波动，但不一定构成明确异常。

### 2.7 异常主因

页面会给每个异常点一个主因：

| 主因 | 判定逻辑 | 含义 |
|---|---|---|
| 负值/倒挂 | `price < 0` | 信号出现负值或业务倒挂 |
| 小时突变 | `z_change` 最大 | 相邻小时变化过快 |
| 短窗残差 | `z_residual` 最大 | 偏离最近 24 小时状态 |
| 季节模式偏离 | `z_pattern` 最大 | 偏离同月份同小时常见模式 |
| 幅值越限 | 其他情况 | 信号本身过高或过低 |

通俗解释：主因不是最终故障原因，而是“为什么算法认为它异常”的解释线索。

## 3. 信号分析

信号分析模块回答：

1. 数据是否有周期规律？
2. 不同时间尺度上的波动强弱怎样？
3. 每天的工况可以分成哪些类型？

## 3.1 傅里叶频谱

页面使用 FFT，也就是快速傅里叶变换，将时序信号从时间域转换到频率域。

### 3.1.1 输入序列

先把数据按小时重采样：

```text
hourly_price = resample(price, every 1 hour)
```

缺失值用时间插值补齐：

```text
hourly_price = interpolate(hourly_price)
```

再去均值：

```text
y[t] = hourly_price[t] - mean(hourly_price)
```

通俗解释：去均值是为了让频谱更关注“波动周期”，而不是整体水平高低。

### 3.1.2 汉宁窗

页面使用汉宁窗减少频谱泄漏：

```text
window[t] = hanning_window[t]
y_windowed[t] = y[t] * window[t]
```

通俗解释：时序数据被截成一段来分析时，首尾可能不连续，会影响频谱结果。加窗相当于让首尾更平滑。

### 3.1.3 FFT 功率谱

计算频谱：

```text
Y[f] = FFT(y_windowed)
power[f] = |Y[f]|^2
```

频率转换为周期：

```text
period_hours = 1 / frequency
```

功率转换为 dB：

```text
power_db = 10 * log10(power + 1e-9)
```

页面只保留 2 小时到 45 天之间的周期：

```text
2 <= period_hours <= 24 * 45
```

通俗解释：FFT 可以找出隐藏的周期，例如 24 小时日周期、168 小时周周期。如果换成设备振动数据，频谱可以帮助发现转频、倍频、冲击频率等。

### 3.1.4 主周期识别

页面按功率从高到低取前 8 个周期：

```text
top_periods = sort(power, descending).head(8)
```

通俗解释：主周期越强，说明该周期在信号中越明显。比如电价数据通常会有日周期；设备数据可能会有转速相关周期或运行班次周期。

## 3.2 多尺度残差能量

页面计算 6 个时间尺度的残差能量：

```text
windows = [3, 6, 12, 24, 72, 168] hours
```

对每个窗口：

```text
baseline_w[t] = rolling_mean(price, window=w)
residual_w[t] = price[t] - baseline_w[t]
energy_w = sqrt(mean(residual_w^2))
```

各尺度解释：

| 尺度 | 页面说明 | 通俗解释 |
|---|---|---|
| 3小时 | 瞬时扰动 | 很短时间内的尖峰、冲击 |
| 6小时 | 班次波动 | 半个班次或短时运行变化 |
| 12小时 | 半日模式 | 半天尺度的状态变化 |
| 24小时 | 日周期偏离 | 日内规律是否稳定 |
| 72小时 | 短期工况漂移 | 近几天工况是否漂移 |
| 168小时 | 周周期偏离 | 一周规律是否改变 |

通俗解释：能量越大，说明该尺度上的波动越强。例如 3 小时能量大，说明瞬时冲击多；168 小时能量大，说明周尺度的运行规律变化明显。

## 3.3 日级工况聚类

页面把小时数据聚合成每天一行，然后用 K-means 思路进行工况分群。

### 3.3.1 日级特征

每天计算：

```text
avg_price = mean(price)
max_price = max(price)
min_price = min(price)
volatility = std(price)
anomaly_hours = sum(is_anomaly)
avg_score = mean(diagnostic_score)
negative_hours = sum(price < 0)
peak_to_valley = max_price - min_price
```

通俗解释：每一天被概括成一组“日工况特征”，包括平均水平、波动强度、异常数量、负值数量等。

### 3.3.2 标准化

聚类前对特征标准化：

```text
x_standard = (x - mean(x)) / std(x)
```

通俗解释：不同特征量纲不同，例如价格均值和异常小时数不能直接比较。标准化后，每个特征都变成相近尺度。

### 3.3.3 K-means 聚类

页面使用 NumPy 实现 K-means。

目标是让同一簇内部尽量相似：

```text
minimize sum(||x_i - center_cluster(i)||^2)
```

迭代过程：

1. 初始化若干聚类中心。
2. 每个样本分配到最近的中心。
3. 用簇内样本平均值更新中心。
4. 重复直到分配结果稳定或达到迭代上限。

通俗解释：K-means 会把每天分成几类，例如“低波动基线、常规运行、高负荷波动、异常冲击”等，帮助快速看出哪些日期属于异常工况。

### 3.3.4 工况命名

页面按聚类后的平均异常评分、波动和均值排序，并命名为：

```text
低波动基线
常规运行
高负荷波动
异常冲击
极端扰动
```

通俗解释：这些名字是为了展示和理解方便，并不是人工标注的真实故障类别。真实项目中可以结合检修记录、故障标签和设备专家经验重新命名。

## 4. 模型评估

模型评估模块回答：

1. 用历史数据能否预测当前信号？
2. 预测误差多大？
3. 哪些特征可以扩展到 SVM、随机森林、XGBoost？

当前页面实现的是一个 Ridge 时序预测基线，不依赖 `scikit-learn`，完全用 NumPy 计算。

## 4.1 预测目标

目标变量：

```text
target[t] = price[t]
```

通俗解释：模型尝试根据历史和时间特征预测当前小时的信号值。

## 4.2 预测特征

页面使用以下特征：

| 特征 | 公式 | 含义 |
|---|---|---|
| `lag_1h` | `price[t-1]` | 上一小时的信号值 |
| `lag_24h` | `price[t-24]` | 前一天同小时的信号值 |
| `lag_168h` | `price[t-168]` | 前一周同小时的信号值 |
| `rolling_mean_24h` | `mean(price[t-24], ..., price[t-1])` | 过去 24 小时平均水平 |
| `rolling_std_24h` | `std(price[t-24], ..., price[t-1])` | 过去 24 小时波动程度 |
| `hour_sin/hour_cos` | 小时周期编码 | 表示一天内的周期位置 |
| `week_sin/week_cos` | 星期周期编码 | 表示一周内的周期位置 |
| `month_sin/month_cos` | 月份周期编码 | 表示一年内的季节位置 |

### 4.2.1 周期编码公式

为了表达周期性，页面不用简单的数字 0、1、2，而是用正弦和余弦。

小时周期：

```text
hour_sin = sin(2π * hour / 24)
hour_cos = cos(2π * hour / 24)
```

星期周期：

```text
week_sin = sin(2π * weekday / 7)
week_cos = cos(2π * weekday / 7)
```

月份周期：

```text
month_sin = sin(2π * month / 12)
month_cos = cos(2π * month / 12)
```

通俗解释：23 点和 0 点在时间上很接近，但如果直接用数字表示，23 和 0 看起来差很远。正弦余弦编码可以表达这种“循环相邻”的关系。

## 4.3 训练集和测试集

页面按时间顺序切分：

```text
train = first 78% samples
test = last 22% samples
```

通俗解释：不能随机打乱时间序列，否则模型会提前看到未来信息。按时间切分更接近真实预测场景。

## 4.4 特征标准化

只使用训练集计算均值和标准差：

```text
mu = mean(X_train)
sigma = std(X_train)
X_train_std = (X_train - mu) / sigma
X_test_std = (X_test - mu) / sigma
```

通俗解释：测试集必须使用训练集的标准化参数，避免把未来数据的信息泄露给模型。

## 4.5 Ridge 回归公式

Ridge 回归是在普通线性回归基础上加入 L2 正则项。

模型形式：

```text
y_hat = β0 + β1*x1 + β2*x2 + ... + βn*xn
```

目标函数：

```text
minimize sum((y - Xβ)^2) + α * sum(β_j^2)
```

当前页面：

```text
alpha = 8.0
```

闭式解：

```text
β = (X^T X + αI)^(-1) X^T y
```

其中截距项 `β0` 不参与正则惩罚。

通俗解释：Ridge 回归是一条稳定的预测基线。它不会像复杂模型那样过度贴合噪声，适合做“预测残差是否异常”的基础模型。

## 4.6 评价指标

### 4.6.1 MAE

```text
MAE = mean(|y - y_hat|)
```

通俗解释：平均每个小时预测错多少，越小越好。

### 4.6.2 RMSE

```text
RMSE = sqrt(mean((y - y_hat)^2))
```

通俗解释：对大误差更敏感。如果 RMSE 明显大于 MAE，说明存在少数预测误差特别大的点。

### 4.6.3 R2

```text
R2 = 1 - sum((y - y_hat)^2) / sum((y - mean(y))^2)
```

通俗解释：表示模型解释波动的能力。越接近 1，说明模型越能解释数据变化；接近 0，说明模型和直接预测平均值差不多。

## 4.7 为什么模型评估对故障诊断有用

预测模型不只是为了预测数值，也可以用于异常诊断：

```text
prediction_error[t] = y[t] - y_hat[t]
```

如果某个时间点实际值与预测值差距很大，说明它不符合历史规律和周期规律，可能是异常工况。

通俗解释：如果设备“本来应该这样运行”，但实际运行偏离很多，就值得检查。

| 缺失清洗 | 是否剔除了空模板行和缺失电价行 |
| 日期格式 | 确认按日/月/年解析，避免日期歧义 |

通俗解释：工业诊断项目里，数据质量非常关键。模型结果是否可信，首先取决于时间戳、缺失值、重复值、字段含义是否处理正确。

## 6. 面试中可以这样解释

可以用下面这段话概括项目：

```text
这个项目把国外逐小时电价数据作为工业时序信号样例，完整实现了从数据清洗、时间特征构建、鲁棒异常评分、频域周期识别、多尺度残差能量、日级工况聚类到预测基线建模的流程。

在异常识别上，我没有只用单一阈值，而是综合幅值异常、小时突变、24 小时短窗残差和月份-小时模式偏离，形成可解释的诊断评分。这样既能发现极端值，也能发现相对最近状态或周期模式的异常。

在信号分析上，我使用 FFT 识别主周期，并用不同窗口的残差能量观察短时冲击、日周期偏离和周周期漂移。这个思路可以迁移到振动、电流、温度等设备信号上。

在模型评估上，我用滞后项、滚动统计和周期编码建立 Ridge 时序预测基线，并用 MAE、RMSE、R2 评估预测能力。后续如果有真实故障标签，可以扩展到 SVM、随机森林、XGBoost 做故障分类和特征重要性分析。
```

## 7. 真实设备数据的替换方向

如果后续换成唐智科技相关的真实工业数据，可以这样替换：

| 当前字段 | 真实设备字段 |
|---|---|
| `timestamp` | 采样时间 |
| `price` | 振动、电流、温度、转速、压力等 |
| `is_anomaly` | 故障标签或报警记录 |
| `driver` | 异常解释原因 |
| `regime` | 工况类型 |

可以进一步加入的工业诊断特征：

- RMS 均方根
- 峰峰值
- 峭度
- 偏度
- 峰值因子
- 包络谱
- 阶次分析
- 小波包能量
- 频带能量比
- 轴承故障特征频率

这些特征可以和 SVM、随机森林、XGBoost 结合，用于故障分类、健康度评估和异常趋势预警。
